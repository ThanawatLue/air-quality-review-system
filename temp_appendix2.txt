Module Name

Software Installation

Module Specification: SHA-256 Cryptographic Checksum (MT-02)

Module ID

MT-02

Module Name

SHA-256 Cryptographic Checksum

Description

Calculates the SHA-256 hash of data files and setpoint files according to GAMP 5 standards (UR-DI-03). This ensures that the files have not been modified and guarantees data traceability.



(Fig.2 get_file_hash(…) function in analysis_logic.py)

Module Specification: File Header Detection (MT-03)

Module ID

MT-03

Module Name

File Header Detection

Description

Searches for the header line (which starts with <>Date) in Phase I BAS CSV files to identify the exact starting point for reading the actual data payload.



(Fig.3 find_header(…) function in analysis_logic.py)

Module Specification: Room ID And Point Type Mapping (MT-04)

Module ID

MT-04

Module Name

Room ID And Point Type Mapping

Description

Groups index numbers to find periods where parameter values are continuously out of specification. It checks if the violation lasts longer than the defined min_length (for example, a continuous violation of 25 minutes or more).



(Fig.4 find_continuous_ranges(…) function in analysis_logic.py)



Module Specification: Continuous Range Processing (MT-05)

Module ID

MT-05

Module Name

Continuous Range Processing

Description

Reads the CSV header to determine which data column (e.g., Point_1, Point_2) corresponds to the specific sensor type (TEMP, HUM, PRES) for the target room.





(Fig.5 find_point_mapping(…) function in analysis_logic.py)

Module Specification: File Date Range Extraction (Insight/Info Format) (MT-06)

Module ID

MT-06

Module Name

File Date Range Extraction (Insight/Info Format)

Description

Extracts the start and end dates directly from the contents of the CSV file (Phase I / Insight Format) to accurately determine the temporal boundaries of the dataset being analyzed.



(Fig.6 get_file_date_range(…) function in analysis_logic.py)

Module Specification: Dataframe Preparation (Insight/Info Format) (MT-07)

Module ID

MT-07

Module Name

Dataframe Preparation (Insight/Info Format)

Description

Reads the raw CSV data from the Phase I BAS system. It cleans, structures, and converts the data into a standard DataFrame format, making it ready for analysis.





(Fig.7 prepare_df(…) function in analysis_logic.py)

Module Specification: Pressure Corridor Matching (MT-08)

Module ID

MT-08

Module Name

Pressure Corridor Matching

Description

Identifies and matches the file path of the "Corridor room" that is required for differential pressure comparisons.



(Fig.8 find_compare_path(…) function in analysis_logic.py)

Module Specification: Data Timestamp (MT-09)

Module ID

MT-09

Module Name

Data Timestamp

Description

Extracts the date and time from the filename to establish the starting point for the analysis. It also applies legacy business logic by subtracting one day from the extracted date.



(Fig.9 parse_filename_for_datetime(…) function in analysis_logic.py)

Module Specification: Dataframe Preparation (Desigo Format) (MT-10)

Module ID

MT-10

Module Name

Dataframe Preparation (Desigo Format)

Description

Similar to prepare_df, but designed for Phase II (EMS / Desigo). It merges multiple separate CSV files for different sensors (Temperature: RMT, Humidity: RMH, Pressure: RDP) into a single, unified DataFrame for analysis.



(Fig.10 prepare_df_phase2(…) function in analysis_logic.py)

Module Name
