SECTION 1: BUILD-TIME CONFIGURATION (BASELINE ENVIRONMENT)
To ensure the `AQR_Dashboard_v1.1.0.exe` program can be reproduced exactly the same way in the future (binary reproducibility), the developer must use this specific, controlled environment to build it:

•	Operating System: Windows 10 or Windows 11 (64-bit).
•	Python Version: Version 3.10.11 (The exact programming language version).
•	PyInstaller Version: Version 5.13.0 (The tool used to package the app).
•	Library Manifest: The exact versions of internal software toolkits used:
    •	`Flask==2.2.5` (Used for the web interface)
    •	`waitress==2.1.2` (Used as the hidden web server)
    •	`pandas==1.5.3` (Used for complex data calculations)
    •	`XlsxWriter==3.1.0` (Used to generate the final Excel reports)
•	Build Command: The exact terminal command used to bundle everything into a single, invisible-console application:
    ```powershell
    pyinstaller --onefile --noconsole --add-data "templates;templates" --add-data "static;static" --version-file version_info.txt app.py
    ```

SECTION 2: RUNTIME ENVIRONMENT & PERMISSIONS

2.1 Deployment Folder Structure
When the system is installed on a user's computer, it must be organized in this exact folder structure:
•	`[ROOT]/AQR_Dashboard_v1.1.0.exe` (This is the main program. Its digital fingerprint must match the officially verified baseline).
•	`[ROOT]/data/SetPointLimit.xlsx` (This is the Excel file containing the Acceptable Limits).
•	`[ROOT]/logs/` & `[ROOT]/reports/` (These are the folders where the system automatically saves the Audit Trail and final reports).

SECTION 3: UNC NETWORK PATH SUPPORT
•	The system is designed to access and read data stored on remote network shared drives (known as UNC paths, like `\\SERVER\BAS_Exports\`).
•	Service Account: If the IT department configures this application to run automatically in the background (as a Windows Service), they must ensure that the "Service Account" running it has the proper permission to access those network folders.
